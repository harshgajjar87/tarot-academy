# LinkedIn Post

---

I've been reading tarot for a while now.

Not as a party trick. As a serious study of symbolism, psychology, and pattern recognition.

At some point I had notebooks full of card meanings, color theories, numerology breakdowns, and reading notes. And as a Computer Engineering student, I had one obvious thought:

**What if I turned this into a proper learning platform?**

So I built **Tarot Academy** — a full interactive web app that teaches the complete Rider-Waite tarot system from scratch.

Here's what it covers:

🃏 **All 78 cards** — each with a full picture description, symbol-by-symbol breakdown, upright/reversed meanings, color analysis, and 7 aspects of prediction (Past, Present, Future, Career, Health, Relationship, Daily Guidance)

📜 **History** — from 15th century Italian playing cards to the 1909 Rider-Waite publication that changed everything

🔢 **Numerology** — the mathematical system embedded in every card number

🎨 **Color symbolism** — why Pamela Colman Smith chose every single color she painted

🔮 **6 interactive spread readings** — Single Card, Two Card, Past/Present/Future, Lovers Spread, 12-card Yearly, and the classic Celtic Cross — all with flip animations and real-time card draws

---

**What I actually built (tech side):**

- React 18 + Vite frontend
- Framer Motion for all animations
- Three.js for the interactive 3D hero scene (mouse-reactive antigravity cards)
- Modular data architecture — 5 separate JS files for Major Arcana + 4 suits, merged at runtime so no single file is bloated
- Admin lock system with session-based access control
- Pure CSS glassmorphism UI with custom design system

---

**Where AI came in:**

I used Amazon Q (AWS's AI coding assistant) as a development partner throughout this project.

It helped me scaffold components, debug import chains, generate the written card content (descriptions, symbol meanings, practice Q&As), and speed up the CSS work.

But here's the thing — the tarot knowledge itself came from me. The interpretations, the reading methodology, the educational structure, the decision of what to teach and how — that was all from my own study and practice.

AI was the tool. The knowledge was mine.

That's the part I find genuinely interesting about building with AI right now: it doesn't replace domain expertise. It amplifies it. If I didn't actually know tarot, the AI-generated content would have been generic and shallow. Because I knew the subject deeply, I could direct it, correct it, and shape it into something actually useful.

---

This project sits at an intersection I didn't expect to find myself in — ancient symbolic systems and modern web development. But honestly, both are just different ways of building structured systems for understanding the world.

The deck is a data structure. The spreads are query patterns. The interpretations are context-dependent outputs.

Once I saw it that way, building the app felt completely natural.
